# Database Schema Reference

SQLite database (`companion.db`) implementing a three-layer memory architecture: Episodic, Semantic, and Identity. Additional auxiliary tables support bookmarks, audit logging, heartbeats, coherence monitoring, and a knowledge graph.

---

## Layer 1: Episodic

Raw turn-by-turn record. Never deleted. The permanent log.

### sessions

Top-level container for each conversation session.

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `started_at` | DATETIME | Session start time (default: `CURRENT_TIMESTAMP`) |
| `ended_at` | DATETIME | Session end time (null while session is active) |
| `summary` | TEXT | Post-session summary generated by the companion |
| `mood` | TEXT | Overall session mood label (e.g. `heavy`, `light`, `focused`) |
| `notable` | BOOLEAN | Whether the session was flagged as notable (default: 0) |
| `cli_session_id` | TEXT | UUID of the Claude CLI session used for `--resume` |

### turns

Individual conversation turns within a session.

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `session_id` | INTEGER | Foreign key -> `sessions(id)` |
| `role` | TEXT | Speaker: `will` or `companion` (enforced by CHECK constraint) |
| `content` | TEXT | The text content of the turn |
| `timestamp` | DATETIME | When the turn occurred (default: `CURRENT_TIMESTAMP`) |
| `topic_tags` | TEXT | Comma-separated topic tags |
| `weight` | INTEGER | Importance weight, 1-5 (default: 1, enforced by CHECK constraint) |
| `channel` | TEXT | Communication channel (default: `direct`) |
| `embedding` | BLOB | Vector embedding for semantic search |

**Indexes:**
- `idx_turns_session` on `session_id`
- `idx_turns_timestamp` on `timestamp`

## Layer 2: Semantic

Summarised understanding. Injected at session start for context.

### summaries

Session-level summaries distilling key points from conversations.

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `created_at` | DATETIME | When the summary was created (default: `CURRENT_TIMESTAMP`) |
| `session_id` | INTEGER | Foreign key -> `sessions(id)` |
| `content` | TEXT | The summary text |
| `topics` | TEXT | Comma-separated topic labels |
| `embedding` | BLOB | Vector embedding for semantic search |

**Indexes:**
- `idx_summaries_topics` on `topics`

### threads

Ongoing conversation threads tracked across sessions -- recurring topics, concerns, or projects.

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `name` | TEXT | Short, recognisable thread label |
| `last_updated` | DATETIME | When the thread was last updated |
| `summary` | TEXT | Current state description |
| `status` | TEXT | Thread status: `active`, `dormant`, or `resolved` (enforced by CHECK constraint, default: `active`) |

**Indexes:**
- `idx_threads_status` on `status`

### thread_mentions

Junction table linking turns to threads they reference.

| Column | Type | Description |
|--------|------|-------------|
| `turn_id` | INTEGER | Foreign key -> `turns(id)` |
| `thread_id` | INTEGER | Foreign key -> `threads(id)` |

**Primary key:** composite `(turn_id, thread_id)`

---

## Layer 3: Identity

Persistent model of Will. Updated deliberately, not automatically.

### identity_snapshots

Point-in-time snapshots of the companion's model of Will. Created by explicit identity-level events (e.g. monthly evolution).

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `created_at` | DATETIME | When the snapshot was created (default: `CURRENT_TIMESTAMP`) |
| `trigger` | TEXT | What caused this snapshot (e.g. `monthly_evolution`, `manual`) |
| `content` | TEXT | The full identity model text |

### observations

Individual observations about Will -- patterns, tendencies, preferences, and insights. These accumulate over time and can be retired when they no longer hold.

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `created_at` | DATETIME | When the observation was recorded (default: `CURRENT_TIMESTAMP`) |
| `content` | TEXT | The observation text |
| `source_turn` | INTEGER | Foreign key -> `turns(id)` -- the turn that prompted this observation |
| `incorporated` | BOOLEAN | Whether this observation has been incorporated into an identity snapshot (default: 0) |
| `valid_from` | DATETIME | Bi-temporal: when this observation became true |
| `valid_to` | DATETIME | Bi-temporal: when this observation stopped being true |
| `learned_at` | DATETIME | When the companion learned this (default: `CURRENT_TIMESTAMP`) |
| `expired_at` | DATETIME | When this observation was retired |
| `confidence` | REAL | Confidence score, 0.0--1.0 (default: 0.5) |
| `activation` | REAL | Activation level for memory decay (default: 1.0) |
| `last_accessed` | DATETIME | When this observation was last read/surfaced |
| `embedding` | BLOB | Vector embedding for semantic search |

**Indexes:**
- `idx_observations_inc` on `incorporated`
- `idx_observations_activation` on `activation`

## Auxiliary: Bookmarks

Significant moments the companion marks silently during conversation.

### bookmarks

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `session_id` | INTEGER | Foreign key -> `sessions(id)` |
| `created_at` | DATETIME | When the bookmark was created (default: `CURRENT_TIMESTAMP`) |
| `moment` | TEXT | Description of what made this moment significant |
| `quote` | TEXT | The exact words that mattered (optional) |
| `embedding` | BLOB | Vector embedding for semantic search |

---

## Auxiliary: Audit

### tool_calls

Every tool call the companion makes, logged for review and accountability.

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `session_id` | INTEGER | Foreign key -> `sessions(id)` |
| `timestamp` | DATETIME | When the call was made (default: `CURRENT_TIMESTAMP`) |
| `tool_name` | TEXT | Name of the tool called |
| `input_json` | TEXT | JSON-serialized input arguments |
| `flagged` | BOOLEAN | Whether this call was flagged as suspicious (default: 0) |

**Indexes:**
- `idx_tool_calls_session` on `session_id`
- `idx_tool_calls_flagged` on `flagged`

---

## Auxiliary: Heartbeats

Log of every heartbeat evaluation -- whether or not it resulted in outreach.

### heartbeats

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `timestamp` | DATETIME | When the heartbeat fired (default: `CURRENT_TIMESTAMP`) |
| `decision` | TEXT | What the heartbeat decided to do |
| `reason` | TEXT | Why it made that decision |
| `message` | TEXT | The message sent, if any |

---

## Auxiliary: Coherence Checks

SENTINEL monitor logs -- tracks mid-session degradation checks.

### coherence_checks

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `timestamp` | DATETIME | When the check ran (default: `CURRENT_TIMESTAMP`) |
| `score` | REAL | Coherence score |
| `degraded` | BOOLEAN | Whether degradation was detected |
| `signals` | TEXT | Diagnostic signals observed |
| `action` | TEXT | Action taken in response (default: `none`) |

---

## Auxiliary: Knowledge Graph

Entity extraction and relationship tracking across conversations.

### entities

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `name` | TEXT | Entity name (unique) |
| `entity_type` | TEXT | Category: `person`, `concept`, `place`, `event`, `project` (default: `concept`) |
| `first_seen` | DATETIME | When the entity first appeared (default: `CURRENT_TIMESTAMP`) |
| `last_seen` | DATETIME | When the entity was last mentioned |
| `mention_count` | INTEGER | Total mention count (default: 1) |
| `embedding` | BLOB | Vector embedding |

**Indexes:**
- `idx_entities_name` on `name`

### entity_relations

Relationships between entities.

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Primary key, autoincrement |
| `entity_a` | INTEGER | Foreign key -> `entities(id)` |
| `entity_b` | INTEGER | Foreign key -> `entities(id)` |
| `relation` | TEXT | Relationship type: `discussed_with`, `related_to`, `part_of`, etc. |
| `strength` | REAL | Relationship strength, 0.0--1.0 (default: 0.5) |
| `first_seen` | DATETIME | When the relationship was first observed (default: `CURRENT_TIMESTAMP`) |
| `last_seen` | DATETIME | When the relationship was last reinforced |

**Indexes:**
- `idx_entity_relations_pair` on `(entity_a, entity_b)`

---

## Entity Relationship Diagram

```
sessions 1──* turns
sessions 1──* bookmarks
sessions 1──* tool_calls
sessions 1──* summaries

turns *──* threads          (via thread_mentions)
turns 1──* observations     (via source_turn)

entities *──* entities      (via entity_relations)
```
